# Contributing to VitaDash

First of all, thank you for your interest in contributing to VitaDash!

Whether you're fixing a typo, reporting a bug, improving compatibility, or proposing a new feature, every contribution is appreciated.

## Project Philosophy

Before contributing, please read the project principles in the README.

In short:

* Compatibility comes first.
* Keep things lightweight.
* Every feature must justify its weight.
* Simplicity is preferred over complexity.

If a feature improves the experience on the PS Vita while keeping the project fast and maintainable, it's probably a good fit.

## Ways to Contribute

There are many ways to help:

* Report bugs
* Suggest new features
* Improve documentation
* Improve browser compatibility
* Optimize performance
* Add useful feeds
* Improve accessibility
* Submit pull requests

## Before Opening an Issue

Please check whether the issue has already been reported.

If you're suggesting a new feature, explain:

* Why it would benefit Vita users.
* How it fits the project's philosophy.
* Whether there is a simpler alternative.

## Pull Requests

Please keep pull requests focused.

Small, focused changes are much easier to review than large ones.

Whenever possible:

* Write clear commit messages.
* Keep code readable.
* Avoid unnecessary dependencies.
* Test changes in the PS Vita browser whenever possible.

## Coding Style

Please follow the project's coding standards.

In general:

* Use descriptive names.
* Keep functions small.
* Prefer readability over clever solutions.
* Keep compatibility in mind.

## Project Goals

VitaDash aims to remain:

* Lightweight
* Fast
* Compatible
* Easy to maintain
* Easy to customize

Every contribution should help move the project toward those goals.

## Thank You

Thank you for helping make VitaDash a better experience for the PlayStation Vita community.
